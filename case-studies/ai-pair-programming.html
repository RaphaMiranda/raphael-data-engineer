AI Pair Programming

How I integrate agentic AI systems into data engineering workflows — from pipeline scaffolding to architecture decisions — using Claude Code, Codex, and Antigravity as active collaborators, not just autocomplete.

Mode Agentic Development
Core Tools
smart_toy Claude Code terminal Codex CLI rocket_launch Antigravity

The Approach

AI pair programming in data engineering isn't about generating boilerplate — it's about maintaining architectural intent at speed. The goal is to stay in the decision-making layer while the AI handles the mechanical translation of intent into code, tests, and documentation.

Each tool has a distinct role: Claude Code excels at multi-file reasoning across a codebase, Codex handles terminal-native task execution and scripting, and Antigravity accelerates context-heavy refactoring and exploratory sessions.

Key Principle

AI is a force multiplier, not a replacement for architectural thinking. Every AI-generated pattern is reviewed for correctness, performance, and fit within the existing data model.

Where It Shines

Pipeline scaffolding, schema migrations, test case generation, SQL optimisation, and documentation — tasks that are high-effort but low-ambiguity.

claude-session 
# Scaffold a new Iceberg table pipeline

AI-Assisted Pipeline Workflow

01

Define Intent

Write a clear natural-language spec for the pipeline: source, target, transformation rules, SLA, and any existing patterns to follow.

Human-led
02

AI Scaffolding

Claude Code or Codex generates the pipeline skeleton, data quality suite, and unit tests — following existing patterns in the repo.

Claude Code / Codex
03

Review & Refine

Engineer reviews generated code for correctness, edge cases, and performance. Iterates with the AI on specific sections using targeted prompts.

Human-led + AI-assisted
04

Deploy & Document

AI generates PR descriptions, runbooks, and lineage documentation from the final implementation. CI/CD handles the rest.
