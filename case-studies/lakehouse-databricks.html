Modern Data Lakehouse Architecture with Databricks

Duration 6 Months End-to-End
Core Stack
dataset Databricks water_drop Delta Lake bolt Apache Spark

The Challenge

The client faced a critical failure point: their legacy Hadoop-based infrastructure could no longer handle the 300% YoY data volume growth. ETL pipelines were brittle, and the separation between their data lake (for ML) and data warehouse (for BI) created massive consistency silos.

Legacy Debt

Manual schema management and frequent job failures leading to 48hr data freshness delays.

Modern Req

Real-time ingestion with ACID compliance and unified governance across all personas.

Medallion Architecture

Systemic Flow: Ingestion → Curated → Aggregated

Strategic Trade-offs

compare_arrows

Databricks vs. Snowflake

While Snowflake offers unmatched ease for pure SQL workloads, the client’s heavy reliance on unstructured image data and Spark-based ML models made Databricks the superior architectural choice for a true unified platform.

account_tree

Serverless Compute

Adopted Databricks SQL Serverless to eliminate cluster startup latency for BI users, trading off slightly higher unit costs for massive productivity gains.

pipeline_definition.py 
import dlt
from pyspark.sql.functions import *

@dlt.table
def silver_user_events():
  # Apply schema enforcement
  return (
    dlt.read_stream("bronze_raw")
      .filter(col("valid") == True)
      .dropDuplicates(["event_id"])
  )

@dlt.table
def gold_daily_active():
  return (
    dlt.read("silver_user_events")
      .groupBy(window("ts", "1 day"))
      .count()
  )
